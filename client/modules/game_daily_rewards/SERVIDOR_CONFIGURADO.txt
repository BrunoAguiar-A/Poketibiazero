================================================================================
ESTRUTURA DO HANDLER
================================================================================

O handler funciona assim:

1. REFRESH (Cliente solicita dados)
   ├─ Verifica dia atual (storage)
   ├─ Verifica último claim (storage)
   ├─ Calcula se pode coletar
   └─ Envia dados ao cliente

2. CLAIM (Cliente tenta coletar)
   ├─ Valida se pode coletar
   ├─ Valida se inventário tem espaço
   ├─ Cria item
   ├─ Adiciona ao jogador
   ├─ Atualiza storage (dia + 1)
   ├─ Atualiza último claim
   └─ Envia dados atualizados

================================================================================
STORAGE USADO
================================================================================

STORAGE_DAILY_REWARDS_DAY = 99001
  └─ Armazena o dia atual (1-21)
  └─ Reseta para 1 após dia 21

STORAGE_DAILY_REWARDS_LAST_CLAIM = 99002
  └─ Armazena timestamp do último claim
  └─ Usado para verificar se passou 24h
