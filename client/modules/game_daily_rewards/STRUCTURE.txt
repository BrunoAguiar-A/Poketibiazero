🔄 FLUXO DE FUNCIONAMENTO
================================================================================

1. INICIALIZAÇÃO
   ├─ init() é chamado
   ├─ Interface é carregada (game_daily_rewards.otui)
   ├─ Widgets de recompensa são criados
   ├─ Eventos do jogo são conectados
   └─ Protocolo estendido é registrado

2. QUANDO O JOGADOR CONECTA
   ├─ onGameStart() é chamado
   ├─ requestRewardsData() é executado
   ├─ Servidor responde com dados
   └─ updateRewardsDisplay() atualiza interface

3. QUANDO A JANELA É ABERTA
   ├─ show() é chamado
   ├─ startTimerUpdate() inicia timer
   ├─ Timer atualiza a cada 1 segundo
   └─ updateInfoText() mostra tempo restante

4. QUANDO JOGADOR CLICA EM CLAIM
   ├─ claimReward() é chamado
   ├─ Requisição é enviada ao servidor
   ├─ Servidor valida e envia recompensa
   ├─ onReceiveRewards() processa resposta
   └─ updateRewardsDisplay() atualiza interface

5. QUANDO A JANELA É FECHADA
   ├─ hide() é chamado
   ├─ Timer é parado
   └─ Memória é liberada

================================================================================
📡 PROTOCOLO DE COMUNICAÇÃO
================================================================================

OPCODE: 2

REQUISIÇÕES (Cliente → Servidor):
  • {"action": "refresh"}  - Solicita atualização
  • {"action": "claim"}    - Tenta coletar recompensa

RESPOSTAS (Servidor → Cliente):
  {
    "rewards": {...},      - Dicionário de recompensas
    "currentDay": 5,       - Dia atual (1-21)
    "canClaim": true,      - Se pode coletar hoje
    "nextClaim": 1234567890, - Timestamp da próxima
    "autoShow": false      - Mostrar automaticamente
  }
